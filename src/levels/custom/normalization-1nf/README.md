# 范式分解训练 - 1NF 原子化

## 教程

第一范式（1NF）的核心要求是：**每一列都必须是不可再分的原子值**。  
如果同一行里出现重复组（比如 `course_1 / course_2 / course_3`），说明这张表还没有达到 1NF。

在实际业务中，宽表经常会把“一个实体的多个值”塞到多个列里，这会带来两个问题：

- 新增第 4 门课程时必须改表结构
- 统计课程人数、课程去重等查询会很别扭

更合理的做法是把重复组“拍平”为多行，让每行只表示一个“学生-课程”关系。

## 本关函数依赖

- `student_id -> student_name`
- `(student_id, course_name)` 表示一条选课记录

## 题目

当前表 `student_course_wide` 使用了重复组设计（`course_1~course_3`）。

请你把它分解为 1NF 结构，并输出一个结果集，字段必须是：

- `student_id`
- `student_name`
- `course_name`

要求：

- 每门课程单独一行
- 过滤掉空课程
